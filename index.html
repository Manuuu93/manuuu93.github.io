По взаимодействию с процессингом провайдеры разделяются на: 1. Оффлайн-провайдеров — информационное взаимодействие с такими поставщиками организуется с использованием реестров. Проведение платежей в пользу оффлайн-провайдеров заключается в фиксации платежным процессингом факта оплаты и отправки поставщику реестра оплат. Реестр представляет собой файл установленного формата, содержащий информацию о платежах. Информация о платежах из реестра загружается в информационную систему провайдера. Процессинг рассылает реестры провайдерам в автоматическом режиме согласно расписанию отправки. Подробная информация об информационном взаимодействии с поставщиками услуг с использованием реестров приведена в разделе 7.1.4. 2. Онлайн-провайдеров — информационное взаимодействие с такими поставщиками организуется с использованием платежных шлюзов. Платежный шлюз представляет собой программный модуль, обеспечивающий взаимодействие с поставщиком согласно протоколу взаимодействия. Возможно разрабатывать шлюзы для каждого поставщика отдельно или использовать универсальный шлюз. Универсальный шлюз возможно использовать для поставщиков, работающих по протоколам ОСМП, КиберПлат или другим протоколам, по которым в одном запросе передается один платеж, запрос передается с помощью HTTP-параметров либо в XML-формате в теле запроса. В этом случае для подключения провайдера необходимо настроить конфигурационный файл шлюза. Подробно применение и настройка универсального шлюза описана в руководстве «Руководство по настройке универсального шлюза. Руководство администратора платежной системы».
